## 一、产品概述

### 1.1 产品定位

心灵奇记是一款基于 AI 情感分析的智能日记应用，通过将用户的日常情感与经典文学作品进行智能匹配，帮助用户在文学的世界中找到情感共鸣，实现"以文会心"的独特体验。

### 1.2 目标用户

| 用户群体 | 特征描述 | 核心需求 |
|---------|---------|---------|
| 文艺青年 | 18-35岁，热爱文学，有记录生活习惯 | 寻找情感表达的文学载体 |
| 情感记录者 | 有写日记习惯，希望深度理解自己 | 情感分析与自我认知 |
| 文学爱好者 | 喜欢阅读经典文学，渴望与作品产生连接 | 发现与自己情感契合的文学作品 |
| 心理健康关注者 | 关注情绪管理，希望通过记录调节心态 | 情绪追踪与心理洞察 |

### 1.3 产品价值主张

**核心价值**：让每一次情感记录都能在文学的星空中找到回响

**差异化优势**：
- AI 驱动的多维度情感分析（情绪、意象、场景、主题）
- 经典文学作品智能匹配，建立用户与文学大师的情感连接
- 游戏化的"文学同伴"收集系统，增强用户粘性
- 可视化的情绪追踪时间线，帮助用户洞察情感变化

## 二、功能架构

### 2.1 功能全景图

```
心灵奇记
├── 日记创作模块
│   ├── 文字输入（支持富文本）
│   ├── 图片上传（贴纸式交互）
│   ├── 情绪标签选择
│   └── 预设写作引导
├── AI 分析模块
│   ├── 情感识别（14种情绪类型）
│   ├── 关键词提取
│   ├── 意象识别
│   ├── 场景识别
│   ├── 主题分析
│   ├── 天气类型映射
│   ├── 心理洞察生成
│   └── 图片内容理解（多模态）
├── 文学匹配模块
│   ├── 多维度相似度计算
│   ├── Top3 段落推荐
│   └── AI 生成匹配原因
├── 心灵花园模块
│   ├── 文学同伴收集
│   ├── 同伴成长系统
│   └── 记忆盒（历史日记）
├── 时间线模块
│   ├── 30天情绪曲线
│   └── 日记时间轴浏览
└── 用户系统
    ├── 注册/登录
    └── 个人数据管理
```

### 2.2 核心功能详述

#### 2.2.1 日记创作（首页）

**功能描述**：以3D翻书交互为核心的日记创作体验

**交互流程**：
1. 用户进入首页，看到一本精美的日记本封面
2. 点击封面，书本以3D动画翻开
3. 用户在内页进行日记创作
4. 支持添加图片贴纸（可拖拽定位）
5. 可选择当前情绪标签
6. 点击"保存日志"提交

**功能要点**：
| 功能项 | 说明 |
|-------|------|
| 文字输入 | 最少10字，无上限 |
| 图片上传 | 支持拖拽/点击上传，最多3张 |
| 情绪选择 | 14种预设情绪可选 |
| 预设引导 | 提供写作灵感卡片 |

**情绪类型**：
快乐、悲伤、愤怒、恐惧、惊讶、平静、孤独、感动、困惑、期待、疲惫、充实、矛盾、释然

#### 2.2.2 AI 情感分析

**功能描述**：基于通义千问大模型的多维度情感分析

**分析维度**：
| 维度 | 说明 | 示例 |
|-----|------|-----|
| 情绪识别 | 识别主要情绪及强度 | 悲伤(0.7)、孤独(0.5) |
| 关键词 | 提取核心关键词 | 离别、思念、远方 |
| 意象 | 识别文学意象 | 月亮、雨、窗、路 |
| 场景 | 识别场景环境 | 深夜、车站、家中 |
| 主题 | 归纳情感主题 | 离别、成长、释怀 |
| 天气映射 | 情感→天气可视化 | 晴天/雨天/多云/雷暴/雾天/彩虹 |
| 心理洞察 | 温暖的心理学解读 | 100字以内的洞察文字 |

**图片分析**：
- 支持多模态理解（通义千问VL）
- 分析图片内容、情感氛围
- 将图片信息融入整体情感分析

#### 2.2.3 文学匹配

**功能描述**：根据情感分析结果，智能匹配经典文学段落

**匹配算法**：
```
综合得分 = 情感相似度×0.4 + 关键词重合×0.3 + 意象匹配×0.2 + 场景匹配×0.1
```

**匹配结果**：
- 返回 Top 3 最匹配的文学段落
- 每个匹配包含：段落内容、作者信息、作品名称
- AI 生成个性化的"匹配原因"说明

**文学库**：
| 作者 | 风格标签 |
|-----|---------|
| 李白 | 浪漫 |
| 苏轼 | 豁达 |
| 李清照 | 婉约 |
| 鲁迅 | 深刻 |
| 林徽因 | 灵动 |
| 莎士比亚 | 戏剧 |
| 泰戈尔 | 灵性 |
| 海明威 | 硬朗 |
| 川端康成 | 细腻 |
| 加缪 | 荒诞 |

#### 2.2.4 结果展示页

**功能描述**：展示日记分析结果与文学匹配

**页面结构**：
1. **天气卡片**：根据情感映射的天气可视化
2. **原文引用**：用户日记内容展示
3. **文学共鸣**：Top 3 匹配段落卡片
4. **心理洞察**：AI 生成的心理学解读
5. **花园更新**：新解锁/成长的文学同伴提示

#### 2.2.5 心灵花园

**功能描述**：游戏化的文学同伴收集系统

**核心机制**：
- 每次日记匹配到某作者的作品，该作者的"同伴"会成长
- 首次匹配解锁新同伴
- 同伴有成长阶段（种子→发芽→成长→开花）
- 记忆盒显示历史日记总数

**同伴列表**：
- 共12个同伴槽位
- 10位预设文学家 + 2个待解锁位置
- 每位同伴有专属头像和风格标签

#### 2.2.6 时间线

**功能描述**：情感历程的可视化追踪

**功能组成**：
1. **30天情绪曲线**：ECharts 折线图展示情绪强度变化
2. **日记时间轴**：按时间倒序展示历史日记
   - 天气图标
   - 主要情绪
   - 内容预览
   - 关键词标签

## 三、用户旅程

### 3.1 新用户首次体验

```
注册/登录 → 首页（日记本封面）→ 点击翻开 → 写日记 → 提交
    ↓
结果页（天气+文学匹配+洞察）→ 解锁首个文学同伴 → 访问花园
    ↓
返回首页继续记录 / 查看时间线
```

### 3.2 核心使用场景

**场景1：情绪宣泄**
> 用户深夜失眠，打开心灵奇记记录当下的孤独感。AI 分析后匹配到李清照的词句，用户感到被理解，情绪得到释放。

**场景2：日常记录**
> 用户记录今天的小确幸，系统匹配到泰戈尔的诗句，用户收藏这段文字，同时泰戈尔同伴成长。

**场景3：情绪回顾**
> 用户查看时间线，发现近期情绪波动较大，通过历史日记回顾，获得自我认知。

## 四、非功能需求

### 4.1 性能要求

| 指标 | 要求 |
|-----|------|
| 日记提交响应 | ≤5秒（含AI分析） |
| 页面加载 | ≤2秒 |
| 图片上传 | 支持≤5MB图片 |

### 4.2 兼容性

- 移动端优先（响应式设计）
- 支持深色/浅色主题切换
- 支持主流浏览器（Chrome、Safari、Firefox）

### 4.3 安全性

- 用户数据加密存储
- JWT 身份认证
- 日记内容仅用户本人可见

## 五、技术架构

### 5.1 技术栈

| 层级 | 技术选型 |
|-----|---------|
| 前端 | React 19 + Vite + TailwindCSS + Zustand + Framer Motion |
| 后端 | NestJS + TypeORM + PostgreSQL |
| AI | 通义千问（qwen-max + qwen-vl-max） |
| 向量检索 | pgvector（1024维向量） |
| 部署 | Docker |

### 5.2 数据模型

**核心实体**：
- User（用户）
- Entry（日记）
- Match（匹配记录）
- Passage（文学段落）
- Author（作者）
- Work（作品）
- GardenPlant（花园植物/同伴）

## 八、风险与应对

| 风险 | 应对策略 |
|-----|---------|
| AI 分析不准确 | 持续优化 Prompt，收集用户反馈 |
| 文学库内容有限 | 逐步扩充，支持用户贡献 |
| 用户隐私顾虑 | 明确隐私政策，数据加密 |
| AI 成本过高 | 缓存常见分析，优化调用频率 |
